魔女

  • TypeScript

命令行中编译选项

选项 描述
-w/–watch 在监视模式下运行编译器

tsConfig.json

tsConfig.json 是 ts 编译器的配置文件，ts 编译器将根据 tsConfig.json 中的配置信息对代码进行编译。

根目录下存在 tsConfig.json， terminal 中运行以下命令行，将编译当前目录及子目录中的所有 ts 文件

配置介绍

include

include 指定需要编译的文件路径

  • ** 递归匹配任意子文件夹
  • * 匹配任意文件

exclude

exclude 指定不需要编译的文件路径
exclude 默认配置为 ["node_modules", "bower_components","jspm_packages"]

extends

extends 指定需要继承的配置文件

files

files 具体指明编译的文件列表只有需要编译的文件少时才会用到

compilerOptions

ts 编译器选项

target 要将 ts 编译为 ES 的哪个版本

module 指定使用的模块化规范，【更多关于模块化】

lib 指定项目中要使用的库
Cannot find name ‘console’. Do you need to change your target library? Try changing the ‘lib’ compiler option to include ‘dom’.

outDir 指定编译后文件的存储目录

outFile 将代码合并为一个文件
设置 outFile 扣，所有的全局作用域中的代码会合并到同一个文件中
Only ‘amd’ and ‘system’ modules are supported alongside –outFile.

allowJs 是否编译 js 文件，默认 false 不编译；checkJs 是否对 js 文件进行 ts 语法检查，默认不检查

  • removeComments 编译时移除注释
  • noEmit 不生成编译后的文件 使用频率低，语法检查时使用
  • noEmitOnError 编译错误时不生成编译后文件

noImplicitAny 不允许使用隐匿 any 类型

noImplicitAny 开启

Error:

noImplicitAny 不允许不明确类型的this {noImplicitAny:true}

strictNullChecks 检查语法是否存在空值

TS2531: Object is possibly ‘null’

strict 所有严格检查的总开关
